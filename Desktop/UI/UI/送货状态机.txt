# 项目目标
生成一个楼内服务机器人送/取货系统的完整工程框架，包括：
1. 用户注册与验证（手机号 + 自设取货密码）
2. 取货请求发起、任务生成、任务列表管理
3. 投件/送货流程与状态机
4. 送货员端 UI 输入确认逻辑
5. 车载端状态机与任务执行
6. 消息通知（APP/车载屏幕）
7. 异常处理（取件失败、投件手机号不存在、超时）
8. 可扩展的占位符逻辑，方便后续添加其它请求

# 编程语言与框架
- 后端：Python 3.11
- 数据库：SQLite 或可替换为 MySQL（用于任务列表、用户信息）
- 机器人控制：ROS2 或自定义 API 模拟机器人状态机
- 前端：简单 HTML/JS APP，用于用户发起取货请求和验证
- 车载端：Python + Tkinter / PyQt 模拟送货员端 UI

# 用户模块
- 功能：
  1. 用户注册（手机号 + 取货密码）
  2. 登录
  3. 发起取货请求（门牌号 + 手机尾号或取货密码）
  4. 接收通知（已送达、取货成功、异常提示）

# 送货员端模块
- 功能：
  1. 查看待投件任务列表（任务列表实时更新）
  2. 投件操作：
     - 输入手机号尾号
     - 系统验证是否存在对应取货请求
     - 再点击确认送货按钮
  3. 投件成功后，任务状态更新为“送货中”

# 状态机（车载端）
- 状态：
  1. 初态（Idle / Remote 控制）
     - 可接收：取货请求、占位符请求
     - 不可投件
  2. 待投件（Pending Delivery）
     - 收到取货请求后进入
     - 可接收新的取货请求
     - 投件只能投给已有请求的任务
  3. 送货中（Delivering）
     - 投件确认后进入
     - 可接收新的取货请求
     - 不可投件
  4. 待取货（Await Pickup）
     - 货物送达，等待用户取件
     - 可接收新的取货请求
     - 不可投件
  5. 返回中（Returning）
     - 用户取货后返回初始位置
     - 可接收新的取货请求
     - 不可投件
- 状态切换条件：
  - 初态 → 待投件：接收到取货请求
  - 待投件 → 送货中：送货员确认投件
  - 送货中 → 待取货：货物送达
  - 待取货 → 返回中：用户取货成功
  - 返回中 → 若还有任务，回到初始位置后：待投件状态； 否则：初态   # 这是最新改动！

# 任务管理
- 数据结构：
  - 待投件任务列表（字典/哈希表，key=手机号尾号或取货密码）
  - 正在投递任务（队列）
  - 已完成任务（历史记录）
- 任务操作：
  1. 新增任务：用户发起取货请求
  2. 更新任务：投件确认、送货中状态
  3. 异常处理：手机号不存在、超时、未取件
  4. 消息通知：APP + 车载屏幕

# 异常处理
1. 投件手机号不存在 → 友好提示
2. 投件确认前误操作 → 无操作生效
3. 待取货超时 → 通知用户或返回初始点
4. 取货密码错误 → 拒绝取件
